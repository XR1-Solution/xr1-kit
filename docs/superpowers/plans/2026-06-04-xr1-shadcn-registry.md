# XR1 shadcn Component Registry — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a shadcn-distributable React component registry that implements the XR1 Solution design system (dark TUI/terminal aesthetic), installable via `npx shadcn add @xr1/<name>`, with a Vite preview app and Vitest + Playwright tests.

**Architecture:** A single repo at `~/xr1/xr1-kit`. Component sources live in `registry/xr1/{lib,ui,blocks}`. A `registry.json` manifest + `shadcn build` produce installable JSON in `public/r/`. A Vite + React + Tailwind v4 preview app imports the registry sources via path aliases (resolving consumer-style `@/components/ui/*` and `@/lib/utils` imports) and renders a gallery used for visual review and as the Playwright target. The brand is delivered by a single `xr1-theme` registry item (tokens + fonts + base + keyframes).

**Tech Stack:** React 19, TypeScript, Tailwind CSS v4 (`@tailwindcss/vite`, `@theme inline`), Radix UI, `class-variance-authority`, `clsx` + `tailwind-merge`, `lucide-react`, Vite, Vitest + Testing Library, Playwright, shadcn CLI.

**Reference (read before starting):** design tokens in `~/xr1/design-system/colors_and_type.css`; brand rules in `~/xr1/design-system/README.md`; component specimens in `~/xr1/design-system/preview/comp-*.html`; console patterns in `~/xr1/design-system/ui_kits/console/console.css`.

**Conventions:**
- Item names are prefixed `xr1-` (e.g. `xr1-button`) to avoid collisions with shadcn core names. The `cn` utility reuses shadcn core `utils` via `registryDependencies: ["utils"]` (the CLI creates `@/lib/utils`); a local copy exists only for the dev app.
- Every UI item declares `registryDependencies: ["xr1-theme", "utils", ...siblings]` and its own npm `dependencies`.
- Styling is faithful to the specimens: hairline 1px borders over shadows, square radii (controls `rounded-sm`=2px, cards/inputs `rounded-md`=4px, panels `rounded-lg`=8px), green focus ring, `active:translate-y-px` press, `steps(1)` blinking block cursor, no emoji.

## File Structure (locked decisions)

| Path | Responsibility |
|---|---|
| `package.json` | deps + scripts (`dev`, `build`, `registry:build`, `test`, `test:e2e`, `typecheck`) |
| `tsconfig.json` / `tsconfig.node.json` | TS config + path aliases for dev/typecheck |
| `vite.config.ts` | react + tailwind plugins, alias map, vitest `test` block |
| `index.html`, `src/main.tsx`, `src/App.tsx` | preview app shell + gallery |
| `src/styles/globals.css` | **source of truth** for tokens/@theme/base/keyframes |
| `src/demos/*.tsx` | one demo per component (Playwright/visual target) |
| `registry/xr1/lib/utils.ts` | `cn()` (dev copy) |
| `registry/xr1/ui/*.tsx` | component sources |
| `registry/xr1/blocks/*.tsx` | composed blocks |
| `registry.json` | registry manifest (all items) |
| `components.json` | shadcn config + `@xr1` registry mapping |
| `public/r/*.json` | generated by `shadcn build` (gitignored except built on demand) |
| `tests/setup.ts`, `tests/unit/*.test.tsx`, `tests/e2e/*.spec.ts` | tests |
| `README.md` | install + `@xr1` config + token reference |
